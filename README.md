# Voice Calendar — AI 语音日历

> **"说了就算"** — 用一句话创建、管理你的全部日程。
> 语音是它的主要交互方式，AI 是它的理解引擎。

---

## 为什么做这个产品

人们每天口头约定大量日程，但传统日历 App 的输入路径是：

> 解锁 → 打开应用 → 点「+」→ 填标题 → 选日期 → 选时间 → 填地点 → 保存

至少 8 步操作。开车、做饭、走路时根本不可用。

Voice Calendar 用一句话替代八步操作：只需说"明天下午三点和小张聊项目"，系统自动提取时间、事件并创建日程。

### 市场空白

| 事实 | 说明 |
|------|------|
| 主流竞品不支持中文 | Fantastical、Motion、Morgen 的自然语言解析仅支持英语 |
| 平台助手能力有限 | Siri/小爱只能处理简单指令，无法理解"工作日每天早上九点站会" |
| 语音优先产品稀缺 | 目前仅有 0sec (iOS) 是纯 voice-first 日历，且仅支持英语 |
| 国内无对标产品 | 搜索"语音日历"无专注该品类的独立产品 |

## 目标用户

### 核心用户

**高效能职场人** — 日均 5-8 个会议，日程变动频繁。开车/走路时突然想到要加日程，打字操作繁琐。核心诉求：一句话完成日程录入。

**银发族** — 需要记药时间、社区活动、子女拜访。不会打字或打字很慢，日历 App 界面复杂。核心诉求：像说话一样自然地管理日程。

**学生 / 自由职业者** — 日程来源碎片化（口头约定、临时通知），手动整理耗时。核心诉求：语音快速捕获，智能理解"大后天""下周三四"等模糊表达。

### 次要用户

- **车载 / 运动场景用户** — 免提操作是安全刚需
- **家庭管理者** — 统筹全家日程（孩子课外班、老人就医）

## 产品定位与演进

Voice Calendar 定位为**语音优先的日历管理工具**，当前处于 L2 阶段。

```
L1 语音日历助手（已完成）
  语音 → 结构化事件（CRUD），单轮 + 简单多轮交互
  │    智能解析引擎 + 确认卡片 + 语音播报闭环
  │
L2 时间管理 Agent（当前版本）
  多轮对话 + 上下文记忆 + 冲突检测
  │    "把下午的会都往后推半小时"
  │    智能提醒 + 原生日历导出
  │
L3 个人效率 Agent（远期）
  学习用户习惯 → 主动建议
       "你最近三周都没运动，要不要安排一次？"
```

## 核心特性

### 语音交互

- **按住说话** — 按住麦克风按钮说话，松开识别；短按（< 300ms）切换开关
- **多语言识别** — 支持中文（普通话）、英文、日语，跟随界面语言自动切换
- **实时中间结果** — 说话过程中实时显示识别中的文字
- **语音播报闭环** — 操作完成后语音播报结果，形成"说话→确认→播报"完整闭环
- **错误反馈** — 麦克风权限拒绝、网络错误、未检测到语音均有对应提示

> **技术说明：** 语音识别基于浏览器原生 Web Speech API，需要 Chrome/Edge 浏览器且设备有麦克风。

### AI 自然语言解析

- **LLM 驱动** — 接入 DeepSeek / OpenAI 等大语言模型，理解自然语言指令
- **结构化输出** — 将语音文字解析为 intent + params 结构化数据
- **支持 7 种意图** — 创建、删除、修改、查询事件，确认，取消，无法理解
- **多轮对话** — 支持上下文指代消解："加个会" → "改到下午四点" → "好的"三步完成
- **中文相对时间** — 支持"明天""后天""下周三""工作日每天"等表达
- **歧义处理** — 多匹配时列出候选列表，删除和修改均支持
- **解析卡片可编辑** — 确认前可修改标题、时间等字段，无需重新语音输入

### 多轮对话

- **上下文记忆** — 后端维护会话状态，最近 6 条历史注入 LLM 调用
- **指代消解** — 理解"改到3点"、"算了"、"好的"等上下文指代
- **会话管理** — 前端生成 sessionId 持久化，30 分钟无活动自动清理
- **Token 控制** — 历史消息超 2000 token 时自动截断，避免超限

### 事件冲突检测

- **实时检测** — 创建或修改事件时自动检查时间重叠
- **冲突警告** — 前端显示冲突事件列表（黄色警告样式）
- **替代建议** — 冲突时自动计算并显示最多 3 个可用时间段，点击可直接选用
- **强制创建** — 提示冲突后仍可选择"确认"强制创建

### 智能提醒

- **类型感知** — 根据事件内容自动建议提醒时间：会议 15 分钟、就医 60 分钟、吃药 5 分钟
- **用户覆盖** — 用户在指令中明确指定时间时（"提前半小时提醒"）覆盖默认值
- **提示音效** — 提醒触发时播放 Web Audio 合成提示音
- **浏览器通知** — 弹出系统通知（需用户授权）
- **语音播报** — 提醒触发时语音播报事件信息
- **离线保留** — 无 WebSocket 客户端时不标记已通知，下次连接时重试

### 导出到原生日历

- **单事件导出** — 事件详情弹窗中点击"导出到日历"下载 `.ics` 文件
- **批量导出** — 侧边栏一键导出当前月所有事件
- **提醒保留** — `.ics` 文件包含 VALARM 提醒设置，导入手机日历后原生提醒
- **跨平台** — 支持 Apple Calendar、Google Calendar、Outlook 等主流日历应用

### 日历视图

- **月视图** — 默认视图，显示整月事件概览
- **周视图** — 按小时显示一周时间轴
- **日视图** — 按小时显示当天时间线
- **周起始日本地化** — 中文/日文周一起始，英文周日起始
- **点击联动** — 单击日期切换侧边栏显示，双击打开新建事件弹窗
- **今日高亮** — 当前日期以主题色标记

### 事件管理

- **语音创建** — 语音指令解析后创建，包含标题、时间、提醒
- **手动创建** — 点击日历空白区域弹出表单手动添加
- **删除/修改** — 支持模糊匹配，多匹配时展示候选列表
- **查询事件** — 查询指定日期/范围的事件列表，语音播报结果
- **事件详情** — 点击已有事件查看详情、编辑或删除

### 体验设计

- **深色模式** — 暗色主题为默认，支持 light/dark 切换
- **多语言界面** — 界面文案跟随语言切换（中/英/日）
- **零门槛** — Web 应用，打开浏览器即用，无需注册登录
- **移动端访问** — 手机连同一 WiFi 即可通过局域网 IP 访问
- **环境光效** — 浮动渐变光球动画，玻璃态面板
- **输入校验** — 后端严格校验所有请求参数
- **频率限制** — 语音接口 20 次/分/IP，事件接口 60 次/分/IP

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML / CSS / JavaScript | 原生实现，无框架依赖 |
| 语音识别 | Web Speech API | 浏览器原生，支持中/英/日 |
| 语音播报 | SpeechSynthesis API | 浏览器原生语音合成 |
| 提示音效 | Web Audio API | 合成双音提示音，无需音频文件 |
| 后端 | Node.js + Express 5 | RESTful API + WebSocket |
| 数据库 | SQLite (sql.js) | 纯 WASM 实现，防抖持久化 |
| AI 解析 | OpenAI 兼容格式 API | 可切换 DeepSeek / Qwen / OpenAI |
| 实时通信 | WebSocket (ws) | 提醒推送（30 秒轮询检查） |
| 频率限制 | express-rate-limit | 语音 20/min，事件 60/min |
| 会话存储 | 内存 Map | 30 分钟过期，自动清理 |

### 已知技术限制

| 限制 | 原因 | 影响 |
|------|------|------|
| 语音识别需 Chrome/Edge | Web Speech API 仅 Chromium 内核完整支持 | Safari 部分可用，Firefox 不可用 |
| 不支持方言 | Web Speech API 仅支持标准语言代码 | 仅普通话/标准英语/标准日语 |
| 不支持农历 | 无农历转换库，LLM 对农历日期不可靠 | 农历相关指令无法准确解析 |
| 提醒有 30 秒延迟 | 采用轮询检查机制 | 精确到分钟级别，非秒级 |
| 单用户架构 | SQLite 单文件 + 内存会话 | 不支持多用户并发 |
| 需 HTTPS 部署 | Web Speech API 安全上下文要求 | 本地可用 localhost，部署需 HTTPS |
| 重复事件存储但未展开 | 前端未渲染 recurrence 实例 | 创建重复规则后日历只显示一条 |
| 会话不持久 | 会话状态存储在内存中 | 服务重启后进行中的对话上下文丢失 |

## 快速开始

### 环境要求

- Node.js >= 18
- Chrome 或 Edge 浏览器（语音识别必需）
- 麦克风设备
- 可用的 AI API Key（DeepSeek / OpenAI / Qwen）

### 安装

```bash
git clone <repo-url> voice-calendar
cd voice-calendar
npm install
```

### 配置

```bash
cp .env.example .env
```

编辑 `.env` 文件填入你的 AI API 配置：

```env
AI_API_KEY=your-api-key-here
AI_API_BASE=https://api.deepseek.com/v1
AI_MODEL=deepseek-v4-pro
```

支持的 AI 提供商：

| 提供商 | AI_API_BASE | 测试过的模型 |
|--------|------------|-------------|
| DeepSeek | `https://api.deepseek.com/v1` | deepseek-v4-pro |
| OpenAI | `https://api.openai.com/v1` | gpt-4o-mini |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | qwen-turbo |

### 启动

```bash
node server.js
```

终端会输出：

```
Voice Calendar running at http://localhost:3000
Mobile access:  http://192.168.x.x:3000
```

电脑浏览器打开 `http://localhost:3000`，手机连同一 WiFi 后扫描 `Mobile access` 地址即可使用。

> **部署注意：** 部署到服务器时必须启用 HTTPS，否则浏览器会拒绝麦克风权限和语音识别。

### 使用语音

1. 点击右侧面板的麦克风按钮（短按切换，长按说话）
2. 说出指令
3. 查看解析卡片，可修改字段后确认
4. 操作完成后听到语音播报

### 导出到手机日历

1. 点击任意事件 → 弹窗底部点「导出到日历」→ 下载 `.ics` 文件
2. 手机打开 `.ics` 文件 → 自动导入系统日历
3. 导入后手机会在设定时间原生提醒，无需打开浏览器

## API 接口

### 事件 CRUD

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/events?start=&end=` | 获取日期范围内的事件 |
| `GET` | `/api/events/:id` | 获取单个事件 |
| `POST` | `/api/events` | 创建事件（限流 60/min） |
| `PUT` | `/api/events/:id` | 更新事件（限流 60/min） |
| `DELETE` | `/api/events/:id` | 删除事件（限流 60/min） |

### 语音指令

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/voice/parse` | 解析语音文字 + 查询/歧义/冲突检测（限流 20/min） |
| `POST` | `/api/voice/execute` | 执行已确认的操作（限流 20/min） |

### ICS 导出

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/events/:id/ics` | 导出单个事件为 .ics 文件 |
| `GET` | `/api/events/export/ics?start=&end=` | 批量导出事件为 .ics 文件 |

### 其他

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/settings` | 获取当前配置（Key 脱敏） |
| `POST` | `/api/settings` | 运行时更新配置 |
| `GET` | `/api/health` | 健康检查 |

**请求示例：**

```json
POST /api/voice/parse
{
  "text": "明天下午两点开会",
  "language": "zh-CN",
  "sessionId": "uuid-v4",
  "context": { "today": "2026-05-31T00:00:00Z" }
}
```

**响应示例（含冲突）：**

```json
{
  "success": true,
  "intent": "add_event",
  "params": {
    "title": "开会",
    "start_time": "2026-06-01T14:00:00",
    "end_time": "2026-06-01T15:00:00",
    "reminder_minutes": 15
  },
  "reply": "明天下午2点有冲突，已有"项目评审"占据14:00-15:00",
```

## 配置项

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `3000` | 服务端口 |
| `AI_API_KEY` | - | LLM API 密钥（必填） |
| `AI_API_BASE` | `https://api.openai.com/v1` | API 基础地址 |
| `AI_MODEL` | `gpt-4o-mini` | 模型名称 |
| `DB_PATH` | `./data/calendar.db` | 数据库文件路径 |
| `REMINDER_INTERVAL` | `30` | 提醒检查间隔（秒） |

> 配置也可通过前端设置页面动态修改（运行时生效，重启后恢复 .env 值）。

## 已知问题

| 问题 | 状态 | 说明 |
|------|------|------|
| 重复事件仅存储不展开 | 已知 | 创建"每周一开会"后日历只显示一条记录，不会生成每日实例 |
| 会话不持久 | 已知 | 服务重启后进行中的对话上下文丢失（事件数据不受影响） |
| 不支持农历 | 已知 | 无农历转换库，LLM 对农历日期不可靠 |
